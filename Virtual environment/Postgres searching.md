  my conf files:
	  .pg_service.conf,
	  .my_pgpass


Конечно, вот последовательность поиска файлов конфигурации для PostgreSQL, если есть два файла конфигурации:

1. **Поиск в переменной окружения `PGSERVICEFILE`**:
    
    - PostgreSQL сначала проверяет переменную окружения `PGSERVICEFILE`. Если эта переменная установлена, PostgreSQL ищет файл конфигурации в пути, указанном этой переменной.
2. **Поиск в текущей директории**:
    
    - Затем PostgreSQL ищет файл конфигурации `.pg_service.conf` в текущей директории, из которой запущен клиентский процесс PostgreSQL.
3. **Поиск в домашней директории пользователя**:
    
    - Если файл конфигурации не найден в текущей директории, PostgreSQL ищет его в домашней директории текущего пользователя (обычно `/home/username` в Unix-подобных системах или `C:\Users\username` в Windows).
4. **Поиск в стандартных местах на файловой системе**:
    
    - Затем PostgreSQL проверяет стандартные места на файловой системе, где обычно находятся файлы конфигурации. Эти места могут включать в себя пути, такие как `/etc`, `/usr/local/etc`, `/usr/local/pgsql`, `/usr/local/share`, `/usr/share`, и другие.
5. **Поиск в переменной окружения `PGPASSFILE`**:
    
    - Когда файл `.pg_service.conf` найден, PostgreSQL использует его содержимое для определения параметров подключения. Если в файле конфигурации указан параметр `passfile`, который указывает на другой файл для хранения паролей (например, `.pgpass`), PostgreSQL также ищет этот файл, используя переменную окружения `PGPASSFILE` или стандартные места на файловой системе.
6. **Использование значений по умолчанию**:
    
    - Если ни один из файлов конфигурации не найден или не содержит нужных настроек, PostgreSQL использует значения по умолчанию или настройки, определенные в самой базе данных.
