В общем, `--mount`более явно и многословно. Самая большая разница заключается в том, что `-v`синтаксис объединяет все параметры в одном поле, а синтаксис `--mount` их разделяет. Вот сравнение синтаксиса для каждого флага.

Если вам нужно указать параметры драйвера тома, вы должны использовать `--mount`.

- `-v`или `--volume`: Состоит из трех полей, разделенных двоеточиями ( `:`). Поля должны быть расположены в правильном порядке, и значение каждого поля не сразу очевидно.
    
    - В случае именованных томов первое поле представляет собой имя тома и является уникальным на данном хост-компьютере. Для анонимных томов первое поле опускается.
    - Второе поле — это путь, по которому файл или каталог монтируются в контейнер.
    - Третье поле является необязательным и представляет собой список параметров, разделенных запятыми, например `ro`. Эти варианты обсуждаются ниже.
- `--mount`: Состоит из нескольких пар ключ-значение, разделенных запятыми, каждая из которых состоит из `<key>=<value>`кортежа. Синтаксис `--mount`более подробный, чем `-v`или `--volume`, но порядок ключей не имеет значения, и значение флага легче понять.
    
    - Тип `type`монтирования, который может быть [`bind`](https://docs.docker.com/storage/bind-mounts/), `volume`или [`tmpfs`](https://docs.docker.com/storage/tmpfs/). В этой теме обсуждаются тома, поэтому тип всегда `volume`.
    - Гора `source`. Для именованных томов это имя тома. Для анонимных томов это поле опускается. Может быть указано как `source` или `src`.
    - В качестве значения принимает `destination`путь, по которому файл или каталог монтируется в контейнер. Может быть указано как `destination`, `dst`или `target`.
    - Если этот `readonly`параметр присутствует, привязка монтируется [в контейнер как доступная только для чтения](https://docs.docker.com/storage/volumes/#use-a-read-only-volume) . Может быть указано как `readonly`или `ro`.
    - Опция `volume-opt`, которую можно указать более одного раза, принимает пару ключ-значение, состоящую из имени опции и ее значения.
- [docker CLI reference](https://docs.docker.com/reference/cli/docker/)

Список томов:

```console
$ docker volume ls

local               my-vol
```

Проверьте том:

```console
$ docker volume inspect my-vol
[
    {
        "Driver": "local",
        "Labels": {},
        "Mountpoint": "/var/lib/docker/volumes/my-vol/_data",
        "Name": "my-vol",
        "Options": {},
        "Scope": "local"
    }
]
```

Удаление тома:

```console
$ docker volume rm my-vol
```

Примеры `-v`и `--mount`ниже дают тот же результат.

`--mount`

---

```console
$ docker run -d \
  --name devtest \
  --mount source=myvol2,target=/app \
  nginx:latest
```

 `-v`

---

```console
$ docker run -d \
  --name devtest \
  -v myvol2:/app \
  nginx:latest
```
