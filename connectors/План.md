### WEB 
- Пользователь вводит ПИН/ФИО.
- Web-приложение формирует JSON-запрос:

```
{
"request_id": "12345",   
"pin": "12345678901234",   
"fio": "Иванов Иван Иванович"
}

```
- Отправляет это сообщение в **Kafka Topic** (например, `requests-topic`).

### Kafka (Broker)

- Сообщение попадает в **Topic `requests-topic`**.
- Kafka сохраняет его и ждёт, пока Consumer его прочитает

### Сервер безопасности (Consumer + Producer)

- Сервер безопасности подписан на `requests-topic`.
- Получает сообщение.
- Проверяет авторизацию/права.
- Делает запрос в базу данных (ГРНП).
- Формирует ответ и отправляет его в другой топик (например, `responses-topic`).

```
{
  "request_id": "12345",
  "status": "OK",
  "data": {
    "fio": "Иванов Иван Иванович",
    "birth_date": "1990-01-01",
    "address": "г. Бишкек, ул. Абая 10"
  }
}
```

### WEB

- Web-приложение подписано на `responses-topic`.
- Когда появляется ответ с нужным `request_id`, оно его получает.
- Отображает пользователю результат.
